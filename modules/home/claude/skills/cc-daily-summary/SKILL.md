# Claude Code　Daily Summary Generator

今日のClaude codeのチャットログから技術的なトピックや特筆すべき事項を抽出して１日のサマリーを生成するスキルです。

## 実行フロー

### 1. 今日のセッションの取得

`/Users/takutomori/obsidian/claude-sessions` のディレクトリから今日の日付(yyyy-MM-dd)のものが存在するかチェックする
存在しなければ終了する

存在する場合
`/Users/takutomori/obsidian/claude-sessions/{yyyy-MM-dd}/**/*.md` ファイルを探す

### 2. セッションの内容の分析

今日の日付のセッションが存在する場合全てのmdファイルを取得して内容から以下を抽出する

- 重要な技術トピックやキーワード
- 悩んだことやトークテーマ
- 疑問点やその解説

その際、ただの作業ディスカッションはサマリーからは除いて最後のセクションに作業したことを追加する

### 3. 対話形式で今日学んだ内容を振り返る

今日学んだ内容に関する質問を投げかけ、学習者の考えを引き出したり学習した内容を復習できるようにする
定着に効果的な質問を投げかけるようにする

**重要：この振り返りは自由記述形式で行う（AskUserQuestionは使わない）**

学習者が自分の言葉で考え、表現することで知識の定着を促す。選択式ではなく、オープンエンドな質問を投げかける。

#### 振り返りの進め方

1. 学んだトピックから1つ選び、テキストで質問を投げかける
2. ユーザーの自由記述の回答を待つ
3. 回答に対して具体的なフィードバックを与える（正解/不正解ではなく、理解を深める補足や別の視点を提供）
4. 必要に応じて深掘り質問や関連質問を続ける
5. 2〜3問程度の対話を繰り返す

#### 効果的な質問の種類

**理解確認（自分の言葉で説明させる）**
- 「〇〇について、同僚に説明するとしたらどう説明しますか？」
- 「〇〇の仕組みを図に描くとしたら、どんな要素が登場しますか？」

**比較・対比（違いを考えさせる）**
- 「〇〇と△△はどちらも似たような用途ですが、どんな場面でどちらを選びますか？」
- 「〇〇を使わずに同じことを実現するとしたら、どんな方法がありますか？」

**応用・転移（実践への橋渡し）**
- 「今日学んだ〇〇を、自分の現在のプロジェクトでどう活かせそうですか？」
- 「もし〇〇で問題が起きたら、どこから調査を始めますか？」

**メタ認知（学習プロセスの振り返り）**
- 「今日一番『なるほど』と思ったのはどの部分ですか？」
- 「まだモヤモヤしている部分や、もっと深く知りたい部分はありますか？」

#### フィードバックの方針

- 正誤を断定的に判断しない（「間違いです」ではなく「こういう観点もありますね」）
- 学習者の回答を起点に、より深い理解につながる情報を追加する
- 実務での具体例やエッジケースを補足する
- 次のステップや関連トピックを示唆する

### 4. サマリーの生成

2.3で取得した内容をもとに要約を作成

### 5. サマリーの保存

今日のセッションディレクトリに `summary.md` を生成して生成したサマリーを保存する
